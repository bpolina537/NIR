## О проекте

«Атмосфера» — учебный интернет-магазин товаров для дома и декора. Проект объединяет клиентскую часть сайта, серверную обработку данных и средства автоматизации учета товаров, заказов и продаж.

Система предназначена для демонстрации навыков адаптивной верстки, программирования на JavaScript и PHP, работы с SQL, внешними сервисами, REST API и системой контроля версий Git. Проект выполняется в учебном объеме и не является коммерческим релизом.

## Основные страницы магазина

- **Главная страница** — информация о магазине, подборки товаров, преимущества и акции.
- **Каталог** — список товаров с ценами, остатками и средствами фильтрации.
- **Карточка товара** — фотография, описание, цена, характеристики и возможность оформить заказ.
- **Оформление заказа** — контактные данные покупателя, адрес доставки, карта и проверка формы.

## Автоматизируемые процессы

- хранение и получение сведений о товарах;
- учет цен, количества и товарных остатков;
- расчет итоговой стоимости товарных позиций;
- фильтрация и поиск товаров;
- оформление и сохранение заказов;
- расчет стоимости доставки;
- учет посещений сайта;
- хранение сообщений гостевой книги;
- предоставление доступа к товарам через REST API.

## Запуск проекта

### Требования

- PhpStorm;
- Docker Desktop с запущенным Docker Engine;
- Git.

Устанавливать PHP и MariaDB непосредственно в Windows не требуется: они запускаются в Docker-контейнерах.

### Настройка в PhpStorm

1. Клонировать репозиторий и открыть его корневую папку в PhpStorm.
2. Открыть встроенный терминал PhpStorm.
3. Создать локальный файл настроек:

   ```powershell
   Copy-Item .env.example .env
   ```

4. В `.env` указать логин и пароль учебного сервиса в переменных `EXERCISE_LOGIN` и `EXERCISE_PASSWORD`. Файл `.env` исключён из Git и не публикуется на GitHub.
5. Собрать и запустить проект:

   ```powershell
   docker compose up -d --build
   ```

6. Проверить контейнеры:

   ```powershell
   docker compose ps
   ```

Должны работать два сервиса: `web` (PHP 8.2 + Apache) и `db` (MariaDB 11.4).

### Адреса

- магазин: [http://localhost:8080/](http://localhost:8080/);
- технические задания: [http://localhost:8080/admin.php](http://localhost:8080/admin.php);
- корзина: [http://localhost:8080/cart.php](http://localhost:8080/cart.php);
- REST API: [http://localhost:8080/api/products/](http://localhost:8080/api/products/).

Если порт 8080 занят, изменить `APP_PORT` в `.env`, например на `8081`.

### Открытие на мобильном устройстве

Компьютер и телефон должны находиться в одной Wi-Fi-сети. Узнать IPv4-адрес компьютера:

```powershell
ipconfig
```

В строке адаптера Wi-Fi найти `IPv4 Address` и открыть на телефоне `http://IP-АДРЕС:8080/`. Например: `http://192.168.0.233:8080/`. Адрес `localhost` на телефоне использовать нельзя, поскольку он указывает на сам телефон. При запросе брандмауэра Windows необходимо разрешить доступ в частной сети.

### Unit-тесты

Запустить 10 тестов:

```powershell
docker compose exec -T web php /var/www/tests/run.php
```

Успешный результат: `10/10 тестов пройдено`.

### Остановка

```powershell
docker compose down
```

Данные MariaDB сохраняются в Docker-томе `nir_database` и не удаляются обычной командой `docker compose down`.
